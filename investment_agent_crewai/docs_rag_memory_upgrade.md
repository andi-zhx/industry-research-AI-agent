# RAG + Memory 升级说明（性能与调试）

## 1. 记忆模块升级

- 基于重要性评分决定是否进入长期记忆（阈值默认 7）。
- 所有记忆写入 `ingest_time` 与 `expires_at`，检索阶段自动过滤过期内容。
- 分类 TTL：
  - fact: 365 天
  - opinion: 90 天
  - conclusion: 180 天
  - report_segment: 730 天

## 2. RAG 升级

- 大文档切块 + 多表示索引：
  - 向量化文本使用“摘要 + 关键词 +关键数字”压缩表示，提高召回效率。
  - 返回结果始终返回完整原文 `raw_content`，保障可引用性。
- 混合检索：向量相似度 + 关键词命中重排。
- RAR 闭环：`query_with_reasoning` 支持“检索 -> 评估 -> 再检索”。

## 3. 同步方案建议（最佳实践）

推荐采用 **混合同步**：
1. **上传即增量更新（主路径）**：保证新文件秒级可检索。
2. **每晚离峰增量重建（兜底路径）**：去重、失效清理、索引健康修复。

该策略兼顾投研场景的实时性和长期索引质量。


## 4. 启动稳定性修复（Windows/Chroma Panic）

- 修复了 `chromadb.PersistentClient(...)` 在模块导入阶段直接初始化导致的启动崩溃问题。
- 改为 **lazy init（延迟初始化）**：只有首次真正查询/入库时才创建 client/collection。
- 当本地 Chroma 索引损坏触发 `pyo3_runtime.PanicException` 时，系统会自动将旧目录重命名为 `*_corrupted_时间戳` 并重建新库，避免应用直接退出。
- 同样对 Memory 向量库增加了容错重建逻辑。
